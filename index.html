Children's Privacy (COPPA)

Parental Consent

A parent or legal guardian must create the family account using their own email address and password (or Apple Sign-In). Only authenticated parents can create child profiles. By creating a child profile, the parent provides verifiable consent for that child's use of the App. Parents can review, modify, or delete their child's data at any time through the App's Settings screen.

Minimum Age

Children under 13 (or under 16 in jurisdictions where applicable, such as the EU/EEA under GDPR) may only use CrushIt through a parent-managed child profile. Children cannot create accounts independently. If we learn that a child has created an account without parental consent, we will delete it promptly.

What We Collect From Kids

  • Display name and username
  • Encrypted PIN (bcrypt hash — we never see the plaintext)
  • Optional avatar emoji and color theme
  • Tasks, points, rewards, streaks, and achievements

What We Never Collect From Kids

  • Real full name, date of birth, or government ID
  • Location or GPS data
  • Contacts or address book
  • Advertising identifiers

Parental Rights

  • Review all data we hold on your child
  • Correct or update your child's information
  • Delete your child's profile and all associated data
  • Withdraw consent and stop data collection
  • Export your family's data as JSON

Data Isolation & Security

  • Family isolation — each family's data is completely isolated at the database level. Families cannot see each other's information.
  • Encryption in transit — all data is transmitted over TLS.
  • Password hashing — bcrypt with automatic salting. We never store plaintext passwords or PINs.
  • Row-Level Security — PostgreSQL RLS policies enforce access control on every query.
  • Signed URLs — task proof photos use time-limited signed URLs (24-hour expiry) and are never publicly accessible.
  • Audit logging — all point mutations and approvals are logged for integrity.
